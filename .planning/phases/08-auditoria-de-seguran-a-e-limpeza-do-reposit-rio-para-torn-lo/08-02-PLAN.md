  <files>package.json, .secretlintrc.json, .husky/pre-commit</files>
  <read_first>
    - package.json (ver scripts e devDependencies atuais)
    - .planning/phases/08-auditoria-de-seguran-a-e-limpeza-do-reposit-rio-para-torn-lo/08-RESEARCH.md (secao "Configuracao do secretlint" e "Hook pre-commit com husky")
  </read_first>
  <action>
1. Instalar secretlint e preset como devDependencies na raiz do monorepo:
   ```bash
   npm install --save-dev secretlint @secretlint/secretlint-rule-preset-recommend
   ```

2. Criar `.secretlintrc.json` na raiz com o conteudo:
   ```json
   {
     "rules": [
       {
         "id": "@secretlint/secretlint-rule-preset-recommend"
       }
     ]
   }
   ```

3. Instalar husky como devDependency e inicializar:
   ```bash
   npm install --save-dev husky
   npx husky init
   ```
   Isso cria o diretorio `.husky/` e configura o prepare script no package.json.

4. Criar o hook pre-commit em `.husky/pre-commit` com conteudo:
   ```bash
   npx secretlint --secretlintrcFilePath .secretlintrc.json $(git diff --cached --name-only --diff-filter=ACMR)
   ```
   IMPORTANTE: Varrer apenas arquivos staged (nao todo o codebase) para performance.
   Se nao houver arquivos staged, o hook deve sair com sucesso (comportamento padrao do secretlint quando nao recebe arquivos).

5. Adicionar script `scan:secrets` no package.json raiz dentro de "scripts":
   ```json
   "scan:secrets": "secretlint \"**/*\""
   ```

6. Executar a varredura completa para validar que o codebase esta limpo:
   ```bash
   npx secretlint "**/*"
   ```
   Deve retornar sem alertas. Se houver falsos positivos (ex: valores placeholder no .env.example), configurar allowlist no .secretlintrc.json.

7. Verificar que o hook funciona fazendo um dry-run:
   ```bash
   echo "test" > /tmp/test-secret.txt
   npx secretlint /tmp/test-secret.txt
   ```
