## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge internal padding |
| sm | 8px | Inline element spacing (`gap-2`), card inner tight rows |
| md | 16px | Default element spacing (`gap-4`), card padding (`p-4`) |
| lg | 24px | Section gaps (`gap-6`), card header/content padding |
| xl | 32px | Layout column gaps (`gap-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical padding (desktop) |

Exceptions:
- Touch targets for card-selection buttons (influence card "Perder esta"): minimum 44px height to meet accessibility tap target requirement.
- Player avatar circles: 32px (`h-8 w-8`) — established by lobby, reused in game.

Source: Established by existing lobby page patterns. Exceptions: WCAG 2.5.5 touch target standard.

## Interaction Contract

### Action Buttons (ActionBar)

| State | Condition | Visual |
|-------|-----------|--------|
| Active — all enabled | `phase === AWAITING_ACTION` AND `activePlayerId === myId` AND `coins < 10` | Default `variant="default"` (primary fill) |
| Forced Coup | `phase === AWAITING_ACTION` AND `activePlayerId === myId` AND `coins >= 10` | Income + Foreign Aid: `disabled` with `opacity-50`; Coup: `variant="default"` + notice text "Você deve Golpear" above bar |
| Not my turn | `activePlayerId !== myId` | All buttons `disabled` with `opacity-50`; no notice text |
| Waiting phase | `phase !== AWAITING_ACTION` | All buttons `disabled` |

### Coup Target Selection

1. Player clicks "Golpe" button — `phase` transitions to `AWAITING_COUP_TARGET` locally (optimistic) until server confirms.
2. `<CoupTargetSelector>` replaces `<ActionBar>` with "Selecionar alvo para o Golpe" heading.
3. Each alive, non-self player in `<PlayerPanel>` gains a clickable highlight (`ring-2 ring-destructive cursor-pointer`).
4. Clicking a target emits `GAME_ACTION: { type: "COUP", playerId, targetId }`.
5. Cancel affordance: "Cancelar Golpe" button returns to normal `<ActionBar>` (only valid before server confirms).

### Influence Loss Card Selection

1. Triggered when server sends `STATE_UPDATE` with `phase === RESOLVING_ACTION` and the pending `LOSE_INFLUENCE` target is `myId`.
2. `<InfluenceCardSelector>` renders inline in `<PlayerPanel>` below own player row.
3. Shows own face-down cards as named buttons: card name + "Perder esta" button.
4. No cancel — player must choose. Buttons are full-width, `variant="destructive"`, 44px min height.
5. On click, emits `GAME_ACTION: { type: "LOSE_INFLUENCE", playerId, cardIndex }`.

### Eliminated Player Display

- Player row in `<PlayerPanel>`: `opacity-50`, name text `line-through`, badge shows "Eliminado" with `bg-destructive/20 text-destructive-foreground`.
- Row remains in list (never removed) — preserves turn order legibility.
- Eliminated players receive no action buttons.

### Winner Overlay

- Renders when `phase === GAME_OVER`.
- Full-screen semi-transparent backdrop (`bg-background/90 backdrop-blur-sm`).
- Center card: winner name at 28px semibold, "Vencedor!" label at 14px muted, no action buttons (Post-game scope is Phase 6).

## Accessibility Notes

- All disabled buttons retain `disabled` attribute (not just `opacity-50 pointer-events-none`) for screen reader compatibility.
- Active player entry in `<PlayerPanel>` uses `aria-current="true"`.
- `<InfluenceCardSelector>` buttons use `aria-label="Perder {cardName}"` for screen readers.
- Game log container uses `role="log" aria-live="polite"` so new entries are announced.
- Color is never the sole differentiator — eliminated players also have `line-through` text decoration and "Eliminado" text badge.
