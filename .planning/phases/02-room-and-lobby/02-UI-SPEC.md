## Color

All values are consumed via CSS variables from `globals.css`. Do not use raw hex values in component code.

| Role | CSS Variable | Resolved Value | Usage |
|------|-------------|----------------|-------|
| Dominant (60%) | `--color-background` | `hsl(240 10% 3.9%)` | Page background — deep zinc dark |
| Secondary (30%) | `--color-card` / `--color-secondary` | `hsl(240 10% 3.9%)` / `hsl(240 3.7% 15.9%)` | Card surfaces and muted containers |
| Accent (10%) | `--color-primary` | `hsl(0 0% 98%)` | Reserved: primary CTA button only, focused input ring |
| Ready state (semantic) | `bg-emerald-500` | Tailwind emerald-500 | Player ready badge background — matches ConnectionBadge pattern |
| Destructive | `--color-destructive` | `hsl(0 62.8% 30.6%)` | Not used in this phase (no destructive actions) |
| Muted text | `--color-muted-foreground` | `hsl(240 5% 64.9%)` | Helper text, "Aguardando jogadores...", unready player name |

Accent reserved for: primary CTA button ("Criar Sala", "Entrar na Sala", "Iniciar Jogo") and focused `Input` ring. Never apply accent to decorative elements, cards, or informational badges.

Ready/not-ready color contract:
- Ready: `bg-emerald-500` dot + badge label "Pronto"
- Not ready: `--color-muted-foreground` dot + badge label "Aguardando"

## Screens and Interaction States

### Screen 1: Home Page (`/`)

**Layout:** Single centered column, max-width 400px, vertically centered.

**Elements:**
1. Display heading "Coup Online" — 28px, semibold
2. Username input — placeholder "Seu nome", max 16 characters, validated inline
3. Primary CTA button "Criar Sala" — full-width, disabled until username is 1+ characters
4. Muted helper text "ou entre pelo link de convite enviado por um amigo" — 14px, muted-foreground

**Input validation (inline, no modal):**
- Empty: no error shown until blur
- Too long (>16 chars): input capped at 16 chars via `maxLength`; no error message needed
- Invalid chars: reject silently on keypress (allow only `[a-zA-Z0-9 -]`)

**Username persistence:** On successful room creation, save to `localStorage` key `coup_player_name`. On page load, pre-fill input from `localStorage` if present.

### Screen 2: Lobby Page (`/room/[roomId]`)

**Sub-state A: Username entry (joining player, no name set)**

Layout: Same centered column as home, max-width 400px.

Elements:
1. Heading "Entrar na Sala" — 20px, semibold
2. Room ID shown as muted label "Sala: {roomId}" — 14px, muted-foreground
3. Username input — placeholder "Seu nome", same validation as home screen
4. Primary CTA button "Entrar na Sala" — full-width, disabled until username 1+ chars

**Sub-state B: Active lobby (player has a name)**

Layout: Centered column, max-width 480px. Two stacked Card sections.

**Card 1: Invite link**
- Section label "Convite" — 14px, semibold, muted-foreground
- Invite URL displayed as monospace text, truncated with ellipsis if too long, 14px
- Button "Copiar link" — secondary variant, icon `Copy` (lucide), full-width
- On copy success: button label changes to "Copiado!" for exactly 2 seconds, then reverts
- Copy failure fallback: display the URL as selectable text with prompt "Copie o link manualmente"

**Card 2: Jogadores ({count}/6)**
- Card header: "Jogadores ({current}/{max})" — 16px, semibold
- Player list: one row per player, ordered by join time
  - Player row layout: `[avatar circle] [name] [ready badge]`
  - Avatar circle: 32px diameter, `bg-secondary`, white initial letter, 14px semibold
  - Name: 16px regular
  - Ready badge: `Badge` component — "Pronto" with emerald-500 background when ready; "Aguardando" with muted background when not ready
  - Host player row: append "(anfitrião)" in muted text after name at 14px

**Host-only element:** "Iniciar Jogo" primary button below Card 2.
- Disabled state (default): grayed out (`opacity-50`), cursor not-allowed, visible from the moment host enters lobby
- Enabled state: full primary styling — only when ALL players are ready AND player count >= 2
- Disabled tooltip (visible on hover): "Aguarde todos os jogadores ficarem prontos"

**Non-host players:** "Iniciar Jogo" button is not rendered.

**Ready toggle:**
- Each player sees their own "Estou Pronto!" button below the player list
- On click: button label changes to "Cancelar Prontidão", button variant changes to `outline`
- Toggle is idempotent (click again to unready)
- Other players' ready status updates in real time via WebSocket without page reload

**Room full state (6/6 players):**
- New join attempt: show inline error "Sala cheia. Máximo de 6 jogadores." at top of username entry form
- Do not render the username entry form if room is full and user has no session in this room

## States and Edge Cases

| State | What the UI shows |
|-------|-------------------|
| Connecting (WebSocket not yet open) | `ConnectionBadge` shows "Conectando..." (existing behavior) |
| Room not found (invalid roomId) | Full-page centered message: "Sala não encontrada." + button "Voltar ao início" |
| Room full on join attempt | Inline error below username input: "Sala cheia. Máximo de 6 jogadores." |
| Host leaves before game starts | Deferred to Phase 6 — no UI handling in this phase |
| Only 1 player present | "Iniciar Jogo" disabled; tooltip "Aguarde ao menos 2 jogadores" |
