  <action>Run `npm run dev` from repo root to start both services, then follow the verification steps below.</action>
  <what-built>
    Complete room and lobby system:
    - Home page with username input and "Criar Sala" button
    - POST /api/rooms backend endpoint creating rooms with 8-char IDs
    - Dynamic lobby page at /room/[roomId] with invite link, player list, ready check, start button
    - Real-time LOBBY_UPDATE broadcasts via Socket.IO
    - Room capacity enforcement (max 6 players)
    - Game start trigger (all ready + >= 2 players + host click)
  </what-built>
  <how-to-verify>
    Run both services:
    ```
    npm run dev
    ```
    (starts frontend at http://localhost:3000 and backend at http://localhost:3001)

    **Test 1 — Room creation (ROOM-01):**
    1. Open http://localhost:3000 in Browser A
    2. Type a name (e.g. "Alice") — verify "Criar Sala" button enables
    3. Click "Criar Sala"
    4. Verify redirect to /room/[8-char-alphanumeric-id]
    5. Verify lobby shows Card "Convite" with the current URL
    6. Click "Copiar link" — verify button shows "Copiado!" for ~2 seconds then reverts

    **Test 2 — Player join (ROOM-02):**
    1. Copy the invite URL from Test 1
    2. Open it in Browser B (or incognito window)
    3. Verify "Entrar na Sala" form appears with "Sala: [roomId]" label
    4. Type a name (e.g. "Bob") and click "Entrar na Sala"
    5. Verify "Bob" appears in Browser A's player list without page reload
    6. Verify "Alice" also appears in Browser B's player list

    **Test 3 — Ready check (ROOM-05):**
    1. In Browser A (Alice), click "Estou Pronto!"
    2. Verify badge next to "Alice" changes to "Pronto" (emerald green) in BOTH browsers
    3. Verify "Iniciar Jogo" button remains disabled (Bob not ready yet)
    4. In Browser B (Bob), click "Estou Pronto!"
    5. Verify badge next to "Bob" changes to "Pronto" in BOTH browsers
    6. Verify "Iniciar Jogo" button becomes active in Browser A (host only)
    7. Verify "Iniciar Jogo" button is NOT rendered in Browser B

    **Test 4 — Game start (ROOM-06):**
    1. In Browser A (Alice, host), click "Iniciar Jogo"
    2. Verify no error appears (game started successfully)
    3. Check browser console for any Socket.IO errors

    **Test 5 — Room full rejection (ROOM-03):**
    1. Create a new room with a fresh browser session
    2. Join with 5 more browser tabs/windows (total 6 players)
    3. Attempt to join with a 7th window
    4. Verify error "Sala cheia. Máximo de 6 jogadores." appears

    **Test 6 — Username persistence:**
    1. Reload Browser A (or navigate back to home then return)
    2. Verify the username "Alice" is pre-filled in the input field

    **Test 7 — Room not found:**
    1. Navigate to http://localhost:3000/room/invalid123
    2. Verify "Sala não encontrada." message appears with "Voltar ao início" button
