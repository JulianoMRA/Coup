User must perform the following deployment steps. Claude cannot do these because Railway and Vercel CLIs are not installed and dashboard setup requires account authentication.

**Step 1 — Deploy backend to Railway:**
1. Go to https://railway.com/dashboard
2. Click "New Project" -> "Deploy from GitHub Repo"
3. Select the Coup repository
4. Set Root Directory to `apps/backend` (in Service Settings)
5. Railway will auto-detect Node.js and use `railway.json` for build/start commands
6. Set environment variables in the Variables tab:
   - `NODE_ENV` = `production`
   - `FRONTEND_URL` = (leave blank for now — set after Vercel deploy)
7. Deploy. Note the generated URL (e.g., `https://coup-xxx.up.railway.app`)

**Step 2 — Deploy frontend to Vercel:**
1. Go to https://vercel.com/dashboard
2. Click "Add New..." -> "Project" -> Import the Coup GitHub repo
3. Set Root Directory to `apps/frontend`
4. Framework Preset should auto-detect Next.js
5. Add environment variable:
   - `NEXT_PUBLIC_BACKEND_URL` = the Railway URL from Step 1 (e.g., `https://coup-xxx.up.railway.app`)
6. Deploy. Note the generated URL (e.g., `https://coup-xxx.vercel.app`)

**Step 3 — Update Railway CORS:**
1. Go back to Railway dashboard -> your service -> Variables
2. Set `FRONTEND_URL` = the Vercel URL from Step 2 (e.g., `https://coup-xxx.vercel.app`)
3. Railway will auto-redeploy with the updated CORS config

**Step 4 — Verify connectivity:**
1. Visit the Vercel URL in a browser
2. Open browser console — confirm no CORS errors
3. The connection badge should show "Conectado" (green)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public internet -> Railway | Backend exposed on public URL; Express + Socket.IO accept connections |
| Public internet -> Vercel | Frontend served as static + SSR on public URL |
| Vercel -> Railway (CORS) | Cross-origin WebSocket and HTTP requests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Spoofing | Railway public endpoint | mitigate | CORS restricted to single Vercel domain via FRONTEND_URL env var; no wildcard |
| T-07-06 | Denial of Service | Railway backend | accept | Friend-group app with max 6 concurrent players per room; Railway provides basic infrastructure protection |
| T-07-07 | Information Disclosure | Vercel env vars | mitigate | Only NEXT_PUBLIC_BACKEND_URL is public (by design — it's the backend URL); no secrets in frontend env vars |
| T-07-08 | Tampering | Socket.IO messages | mitigate | All game actions validated server-side by FSM processAction; invalid actions rejected with error |
</threat_model>
