## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline micro-spacing |
| sm | 8px | Button inner padding (vertical), list item gaps |
| md | 16px | Default element spacing, card content padding, header px |
| lg | 24px | Section padding, action bar px |
| xl | 32px | Layout gaps between major panels |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Action bar fixed-bottom height: auto (py-3 = 12px top/bottom); existing value retained
- Touch targets for all interactive buttons: minimum 44px tall (already met by shadcn Button default height of h-10 = 40px + border — verify on mobile; use `min-h-[44px]` override if needed)
- Header height: 56px (h-14) — existing value retained

Source: `game-board.tsx`, `action-bar.tsx` — existing layout confirmed; no changes to spacing scale needed.

## Responsive Layout Contract

| Breakpoint | GameLog behavior | Layout |
|------------|-----------------|--------|
| `< md` (mobile, < 768px) | Hidden by default; toggled via "Ver Log" / "Ocultar Log" button | Single column (`flex-col`) |
| `md:` and above (desktop, >= 768px) | Always visible; toggle button hidden | Two columns (`flex-col md:flex-row`): left panel 288px (w-72), right panel flex-1 |

Implementation note: The toggle `<Button>` and conditional `{showMobileLog && <GameLog>}` must be wrapped in a `<div className="flex-1">` that on `md:` always renders `<GameLog>` without the button. Use a structure like:

```
<div className="flex-1">
  {/* Mobile toggle — hidden on md+ */}
  <div className="md:hidden">
    <Button ...>Ver Log / Ocultar Log</Button>
    {showMobileLog && <GameLog />}
  </div>
  {/* Desktop always-visible — hidden on mobile */}
  <div className="hidden md:block">
    <GameLog />
  </div>
</div>
```

Source: CONTEXT.md `## Implementation Decisions — Mobile Layout`.
