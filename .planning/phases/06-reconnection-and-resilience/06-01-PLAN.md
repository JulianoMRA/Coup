<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> REJOIN_ROOM | Untrusted roomId from client; playerId from auth (trusted) |
| client -> REMATCH | Untrusted roomId from client; playerId from auth (trusted) |
| cleanup -> rooms Map | Internal server-only; no external input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing | REJOIN_ROOM handler | mitigate | Verify playerId (from socket.handshake.auth) exists in room.players before re-joining channel; never trust playerId from event payload |
| T-06-02 | Tampering | REMATCH handler | mitigate | Check game.phase === GAME_OVER before allowing rematch; verify caller is in room.players |
| T-06-03 | DoS | REMATCH spam | accept | Subsequent REMATCH calls after first succeed are harmless (initGame re-called, no accumulation); rate limiting deferred to v2 |
| T-06-04 | Information Disclosure | disconnectedPlayers field | accept | Contains only playerIds which are already public in all STATE_UPDATE projections; no secret data |
| T-06-05 | Tampering | Cleanup scanner | mitigate | Never delete rooms with status "IN_GAME"; check io.sockets.adapter.rooms for connected sockets before deletion |
| T-06-06 | Elevation of Privilege | REJOIN_ROOM bypassing joinRoom | mitigate | REJOIN_ROOM only calls socket.join(roomId) for players already in room.players; does not add new players |
</threat_model>
