## States and Interactions

### Reconnect Loading State
- Trigger: player refreshes mid-game; `game === null` while `REJOIN_ROOM` is in-flight.
- Visual: existing GameBoard null-guard in `room/[roomId]/page.tsx` ‚Äî no new UI needed.
- Recovery: `STATE_UPDATE` arrives from backend; game renders normally.

### Disconnected Player Indicator
- Trigger: backend pushes `STATE_UPDATE` with `disconnectedPlayers` array containing that player's ID.
- Visual: `(desconectado)` inline span appears next to player name in PlayerPanel row.
- Recovery: span disappears when player reconnects (backend removes from `disconnectedPlayers`, pushes new `STATE_UPDATE`).
- Eliminated players who are also disconnected: show both "Eliminado" badge AND "(desconectado)" span ‚Äî they are not mutually exclusive.

### Rematch Button
- Trigger: `game.phase === GamePhase.GAME_OVER` ‚Äî WinnerOverlay is visible.
- Visual: full-width Button variant `default` labeled "Revanche", `mt-6` below winner name inside the WinnerOverlay card.
- Interaction: single click emits `REMATCH` socket event. No confirmation, no loading spinner. Button does not disable after click (no async wait in v1).
- Outcome: server responds with `GAME_STARTED` + `STATE_UPDATE`; WinnerOverlay unmounts because `game.phase` changes away from `GAME_OVER`.
