  <name>Task 1: End-to-end verification of full Coup ruleset</name>
  <what-built>
    Complete Phase 5 implementation:
    - Backend CHAL-05 card replacement after proven challenge
    - ActionBar with 7 buttons in 2 rows (3 basic + 4 character actions)
    - ReactionBar with Contestar/Bloquear/Passar for AWAITING_REACTIONS
    - BlockClaimSelector for choosing which card to claim when blocking
    - BlockChallengeBar for AWAITING_BLOCK_CHALLENGE (Contestar bloco / Aceitar bloco)
    - ExchangeSelector for Ambassador exchange (select 2 of 4 cards)
    - GameBoard routing all new components by game phase priority
  </what-built>
  <how-to-verify>
    Start both services: `npm run dev` from repo root.

    **Test 1: Character Actions + Reaction Window**
    1. Open 2 browser tabs, create room, join both players, start game
    2. Active player: verify 7 buttons visible (Renda, Ajuda Externa, Golpe in row 1; Imposto (Duque), Roubar (Capitao), Assassinar (Assassino), Embaixador in row 2)
    3. Click "Imposto (Duque)" — non-active player should see ReactionBar with banner "[Name] declarou Imposto (Duque)" and buttons Contestar / Passar (no Bloquear — Tax is not blockable)
    4. Click Passar on non-active player — action should resolve, active player gains 3 coins

    **Test 2: Challenge Flow**
    5. Active player declares Tax again
    6. Non-active player clicks Contestar
    7. Verify InfluenceCardSelector appears for the appropriate player (the one who needs to lose a card)
    8. Select a card to lose — verify the game continues to next turn

    **Test 3: Block Flow**
    9. Active player declares Ajuda Externa
    10. Non-active player should see Contestar / Bloquear / Passar buttons
    11. Click Bloquear — BlockClaimSelector should appear with "Duque" option
    12. Click Duque — verify BlockChallengeBar appears for active player with "Contestar bloco" / "Aceitar bloco"
    13. Click "Aceitar bloco" — action should be cancelled, turn advances

    **Test 4: Steal with Target**
    14. Active player clicks "Roubar (Capitao)" — target selector should appear
    15. Select a target — ReactionBar should appear for the target with Contestar / Bloquear / Passar (target can block with Captain or Ambassador)

    **Test 5: Assassinate with Target**
    16. Accumulate 3+ coins via Income, then click "Assassinar (Assassino)" — verify disabled if < 3 coins
    17. Select target — ReactionBar appears with Contestar / Bloquear (Contessa) / Passar for target

    **Test 6: Ambassador Exchange**
    18. Active player clicks "Embaixador" — reaction window appears
    19. All non-acting players pass — ExchangeSelector should appear showing 4 card badges
    20. Select exactly 2 cards (verify ring highlight and opacity changes)
    21. Verify "Confirmar Troca" button is disabled until 2 selected, then enabled
    22. Click confirm — verify cards update in "Minhas Cartas" section

    **Test 7: Pending Players Indicator**
    23. Open 3 browser tabs (3 players)
    24. Declare any character action — verify "Aguardando: [name1], [name2]" text appears
    25. One player passes — verify their name disappears from the Aguardando list
  </how-to-verify>
