## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(240 10% 3.9%)` — `bg-background` | Page background, bottom bar shell background |
| Secondary (30%) | `hsl(240 3.7% 15.9%)` — `bg-secondary` / `bg-muted` | Button secondary variant, card badges, muted surfaces |
| Accent (10%) | `hsl(240 4.9% 83.9%)` — `ring` / `ring-primary` | Selected card highlight ring in ExchangeSelector only |
| Destructive | `hsl(0 62.8% 30.6%)` — `bg-destructive` | Destructive action buttons: Golpe, Assassinar, InfluenceCardSelector buttons, ring highlight on CoupTargetSelector targets |

Source: `apps/frontend/src/app/globals.css`

Accent reserved for:
- `ring-2 ring-primary` on selected card badges in ExchangeSelector (exactly 2 cards selected state)

Accent NOT used for:
- General hover states (handled by shadcn default)
- Reaction bar buttons (use `variant="secondary"` or `variant="outline"`)
- Pending player indicators (use `text-muted-foreground`)

Color rules for new components:
- `Contestar` button: `variant="destructive"` — challenging is an aggressive, potentially irreversible act
- `Bloquear` button: `variant="outline"` — blocking is defensive, reversible if challenge wins
- `Passar` button: `variant="secondary"` — passive, no consequence
- `Aceitar bloco` button: `variant="secondary"` — passive acceptance
- `Confirmar Troca` button: `variant="default"` — primary confirm action, enabled only when 2/2 selected
- Unselected cards in ExchangeSelector: `opacity-50`
- Selected cards in ExchangeSelector: `ring-2 ring-primary opacity-100`

Primary focal points: the active player's coin count displayed in the player list, and the primary CTA button in the bottom bar (Passar / Contestar / Confirmar Troca depending on phase). These are the two elements the player's eye must reach first in any game state.

## Interaction Contracts

### ActionBar — Row 2 (Character Actions)

Layout: fixed bottom bar, height increases to ~80px to accommodate 2 rows.

Row 1 (unchanged): Renda | Ajuda Externa | Golpe
Row 2 (new): Imposto (Duque) | Roubar (Capitão) | Assassinar (Assassino) | Embaixador

Button states:
- All row-2 buttons disabled when `!canAct` or `forcedCoup`
- `Assassinar (Assassino)`: additionally disabled when `myCoins < 3`
- `Roubar (Capitão)` and `Assassinar (Assassino)`: on click, open target selector (same pattern as Coup — set `selectingStealTarget` / `selectingAssassinateTarget` state in GameBoard)

Button variants:
- `Imposto (Duque)`: `variant="secondary"`
- `Roubar (Capitão)`: `variant="secondary"`
- `Assassinar (Assassino)`: `variant="destructive"` (costs coins, targets a player)
- `Embaixador`: `variant="secondary"`

### ReactionBar

Structure (top to bottom inside the fixed bar):
1. Banner line: `"[Name] declarou [ActionLabel] ([CardClaim])"` — `text-sm text-muted-foreground text-center`
2. Pending players line (shown to ALL players): `"Aguardando: [name1], [name2]"` — `text-xs text-muted-foreground text-center` — lists only players whose `pendingReactions[id] === "WAITING"`
3. Button row (shown only to the local player when their reaction is pending):
   - `Contestar` (only if action is challengeable by rules): `variant="destructive"`
   - `Bloquear` (only if action is blockable AND local player has a valid blocker card by rules): `variant="outline"`
   - `Passar`: `variant="secondary"`

When local player has already reacted: hide button row, show `"Você passou — aguardando os demais"` in `text-sm text-muted-foreground text-center`.

BlockClaimSelector (inline, replaces button row when "Bloquear" is clicked):
- Prompt: `"Bloquear como:"` — `text-sm font-semibold text-center`
- One button per valid claim card (Duke / Contessa / Captain / Ambassador as applicable): `variant="outline"`, clicking emits BLOCK immediately
- Cancel link: `"Voltar"` as `variant="ghost"` restores the Contestar / Bloquear / Passar row

### BlockChallengeBar

Structure:
1. Banner: `"[BlockerName] bloqueou com [CardClaim] — Contestar?"` — `text-sm font-semibold text-center`
2. Buttons (only for the active player — who initiated the original action):
   - `Contestar bloco`: `variant="destructive"`
   - `Aceitar bloco`: `variant="secondary"`

Non-active players see only the banner plus `"Aguardando [ActivePlayerName] decidir"` — `text-xs text-muted-foreground text-center`.

### ExchangeSelector

Structure:
1. Prompt: `"Escolha 2 cartas para manter:"` — `text-sm font-semibold text-center`
2. Card grid: 4 clickable Badge-style elements in a flex-wrap row (2 hand cards + 2 drawn cards)
   - Each card: `Badge` with `variant="secondary"`, min 44px height, clickable
   - Selected state: add `ring-2 ring-primary` class, remove `opacity-50`
   - Unselected state: add `opacity-50`
   - Already-at-2-selected: clicking an unselected card does nothing (button is non-interactive until deselect)
3. Counter label: `"(N/2 selecionadas)"` — `text-xs text-muted-foreground` — updates on each toggle
4. Confirm button: `"Confirmar Troca (2/2)"` — `variant="default"` — disabled until exactly 2 selected; emits `EXCHANGE_CHOOSE` with `keptIndices`

Other players: no special UI. They wait silently. GameBoard shows default ActionBar (disabled) with `"Aguardando [Name] trocar cartas"` as the muted-foreground label (identical to "waiting your turn").

## Valid Blocker Rules (Claude's Discretion — locked for implementation)

| Action | Who Can Block | Card Claimed |
|--------|--------------|--------------|
| Foreign Aid | Any player | Duke |
| Steal | Target player only | Captain or Ambassador |
| Assassinate | Target player only | Contessa |
| Tax | Nobody (Duke claim, challengeable but not blockable) | — |
| Exchange | Nobody (Ambassador claim, challengeable but not blockable) | — |

These rules govern which buttons appear in ReactionBar for each local player.

## Edge Cases

| Scenario | UI Behaviour |
|----------|-------------|
| Only 1 unrevealed card remains when InfluenceCardSelector opens | Render the single card button only; prompt unchanged — single tap selects it without ambiguity |
| Exchange when player has only 1 unrevealed card | Server provides 2 drawn cards; selector shows 1 hand card + 2 drawn (3 total, choose 1 to keep) — counter updates label to "(N/1 selecionada)" |
| Player reacts (Passes) and then their turn comes during same render cycle | GameBoard priority logic ensures ActionBar replaces ReactionBar on phase change |
| Blocker is the same player as the active player | Not possible by rules; BlockClaimSelector never shown to active player |
