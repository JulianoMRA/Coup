# Phase 06.1: UI Redesign — Visual Game Interface - Context

**Gathered:** 2026-04-05
**Status:** Ready for planning
**Source:** Direct user decisions (conversation)

<domain>
## Phase Boundary

Complete visual redesign of the Coup Online interface. The existing UI is functional but bare — plain shadcn components with no visual identity. This phase transforms it into a dark, mysterious card-game aesthetic with a proper game board layout, individual player seats, character-colored cards with icons, and a polished home/lobby experience. No new game mechanics — purely visual and UX improvements.

</domain>

<decisions>
## Implementation Decisions

### Visual Theme
- **Style:** Dark/mysterious — conspiracy theme, dark backgrounds, elegant typography
- **Color palette:** Near-black backgrounds (zinc-950/zinc-900), light text, accent colors reserved for character cards and active states
- **Design system:** Existing shadcn (default/zinc/dark CSS-variables) — extend, do not replace
- **Icons:** Lucide React only — NO emojis anywhere in the interface

### Character Cards
- Each character has a dedicated color and icon:
  - **Duque (Duke):** Roxo / purple — suggested icon: `Crown`
  - **Capitão (Captain):** Azul / blue — suggested icon: `Anchor` or `Shield`
  - **Assassino (Assassin):** Vermelho / red — suggested icon: `Sword` or `Dagger` (use `Skull` if dagger unavailable)
  - **Embaixador (Ambassador):** Verde / green — suggested icon: `Globe` or `Briefcase`
  - **Condessa (Countess):** Rosa / pink — suggested icon: `Gem` or `Sparkles` (use lucide equivalent)
- Cards in hand: visually represent as playing cards (front face with color + icon + name, back face when hidden)
- Revealed cards: visually distinct from unrevealed (greyed out, overlay, or "face down" style)
- Card backs: uniform dark pattern — do not reveal character until revealed in-game

### Player Seats / Game Board
- Game board layout: players arranged as "seats at a table" rather than a flat list
- Each player = an individual card/seat component showing: name, coin count (with coin icon), card count (with card icon), status badges
- Active player seat: highlighted ring or glow effect
- Eliminated player seat: visually distinct (greyed out, opacity reduction, "Eliminado" badge)
- Disconnected player: subtle indicator without disrupting layout
- Coin display: icon (e.g., `Coins` from lucide) + number, not just text

### Layout — Desktop (≥ 768px)
- Left panel: player seats arranged vertically or in a grid
- Center/right: game log always visible
- Bottom: action bar (fixed footer)
- Player's own hand: prominent, near action bar or in a dedicated section

### Layout — Mobile (< 768px)
- Single column, optimized for thumb reach
- Player seats: compact horizontal scroll or condensed cards
- Game log: toggle (Ver Log / Ocultar Log) — already implemented
- Action buttons: large touch targets (min 44px)
- Priority: playable with one hand on mobile

### Action Bar
- Find the best layout — current fixed footer works, improve visual design
- Character action buttons should reflect their character's color where possible
- Forced coup notice: visually prominent

### Home Screen (localhost:3000 / /)
- Minimally polished — consistent with game aesthetic
- Dark background, game logo/title treatment
- Name input + Criar Sala button — clean, not elaborate

### Lobby Screen (/room/[roomId])
- Within the game aesthetic
- Player list with avatar initials, ready status clearly visible
- Invite link copy: clean, not cluttered
- Not as elaborate as the game board but clearly part of the same design system

### What NOT to change
- No animations/transitions (out of scope for this phase)
- No new game mechanics or logic
- No changes to socket events, state management, or backend
- No new npm packages unless strictly necessary (lucide-react already installed)
- Keep existing component file structure — restyle in place

</decisions>
